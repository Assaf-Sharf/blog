### 테스트 용으로 chatGPT로 작성되었음.

C++은 C 언어의 확장으로 시작하여 C의 기능을 계승하면서 추가적인 기능을 제공하는 언어입니다. 이 둘 사이에는 몇 가지 중요한 차이점이 있습니다.

## 1. **객체지향 프로그래밍 (OOP) 지원**
C++은 클래스와 객체 지향 프로그래밍을 지원합니다. 이는 데이터와 관련 함수(메서드)를 하나의 단위로 묶어 캡슐화하고, 상속과 다형성을 통해 코드의 재사용성과 유지보수성을 높일 수 있습니다. 반면 C 언어는 객체지향 프로그래밍을 직접 지원하지 않습니다.

## 2. **템플릿 (Templates)**
C++은 템플릿을 통해 일반화된 데이터 구조나 알고리즘을 작성할 수 있습니다. 이를 통해 코드의 재사용성과 유연성을 높일 수 있습니다. C 언어는 템플릿을 지원하지 않습니다.

## 3. **강력한 라이브러리 지원**
C++은 C 언어의 표준 라이브러리를 포함하여 다양한 라이브러리를 지원합니다. 특히 표준 템플릿 라이브러리(STL)는 유용한 데이터 구조와 알고리즘을 제공하여 개발자의 생산성을 높입니다.

## 4. **예외 처리 (Exception Handling)**
C++은 예외 처리 메커니즘을 지원합니다. 이를 통해 프로그램이 예상치 못한 상황에 대비하여 예외를 처리할 수 있습니다. C 언어는 예외 처리를 지원하지 않습니다.

## 5. **함수 오버로딩 (Function Overloading)**
C++은 함수 오버로딩을 지원합니다. 즉, 동일한 이름의 함수를 여러 번 정의하고, 매개변수의 유형 또는 개수에 따라 다른 동작을 수행할 수 있습니다. C 언어는 함수 오버로딩을 지원하지 않습니다.

## 6. **데이터 은닉성 (Data Encapsulation)**
C++은 클래스 내부에서 데이터 은닉성을 구현할 수 있습니다. 이를 통해 클래스의 구현 세부 정보를 숨기고 외부에 인터페이스만 노출시킴으로써 코드의 안정성과 보안성을 높일 수 있습니다. C 언어는 데이터 은닉성을 직접 지원하지 않습니다.

이러한 차이점들로 인해 C++은 C 언어보다 더욱 강력하고 유연한 언어로 폭넓게 사용되고 있습니다. 그러나 C++은 C 언어의 상위 호환성을 유지하고 있기 때문에 C 코드를 대부분 그대로 사용하면서도 추가적인 기능을 활용할 수 있습니다.
