# HTTP 메서드 - PUT, PATCH, DELETE
## PUT
```http
PUT /members/100 HTTP/1.1
Content-Type: application/json

{
    "username": "hello",
    "age": 20
}
```
- **리소스를 대체**
    - 리소스가 있으면 대체
    - 리소스가 없으면 생성
    - 쉽게 이야기 하면 파일 **복사 - 붙여넣기**와 같음
- **중요! 클라이언트가 리소스를 식별**
    - 클라이언트가 리소스 위치를 알고 URI 지정 (GET의 특징)
    - POST와의 차이점

### 리소스가 있는 경우1
**[클라이언트]**
```http
PUT /members/100 HTTP/1.1
Content-Type: application/json

{
    "username": "old",
    "age": 50
}
```
**[서버]**
`/members/100`
```http
{
    "username": "young",
    "age": 20
}
```

---

### 리소스가 있는 경우2
**[서버]**
`/members/100` (리소스 대체)
```http
{
    "username": "old",
    "age": 50
}
```

---
---

### 리소스가 없는 경우1
**[클라이언트]**
```http
PUT /members/100 HTTP/1.1
Content-Type: application/json

{
    "username": "old",
    "age": 50
}
```
**[서버]**
`/members/100` (이런 리소스는 없음)

---

### 리소스가 없는 경우2
**[서버]**
`/members/100` (신규 리소스 생성)
```http
{
    "username": "old",
    "age": 50
}
```

---
---

### 주의! 리소스를 완전히 대체한다1
**[클라이언트]**
```http
PUT /members/100 HTTP/1.1
Content-Type: application/json

{
    <!-- username 필드 없음 -->
    "age": 50
}
```
**[서버]**
`/members/100`
```http
{
    "username": "young",
    "age": 20
}
```

---

### 주의! 리소스를 완전히 대체한다2
**[서버]**
`/members/100` (리소스 대체)
```http
{
    <!-- username 필드 삭제됨 -->
    "age": 50
}
```

## PATCH
```http
PATCH /members/100 HTTP/1.1
Content-Type: application/json

{
    "age": 50
}
```
- 리소스 **부분 변경**

### 리소스 부분 변경1
**[클라이언트]**
```http
PATCH /members/100 HTTP/1.1
Content-Type: application/json

{
    <!-- username 필드 없음 -->
    "age": 50
}
```
**[서버]**
`/members/100`
```http
{
    "username": "young",
    "age": 20
}
```

---

### 리소스 부분 변경2
**[서버]**
`/members/100` (리소스 부분 변경)
```http
{
    "username": "young",
    "age": 50       <!-- age만 50으로 변경 -->
}
```

<br>

## DELETE
```http
DELETE /members/100 HTTP/1.1
Host: localhost:8080
```
- 리소스 제거

### 리소스 제거1
**[클라이언트]**
```http

Host: localhost:8080
```

**[서버]**
`/members/100`
```http
{
    "username": "young",
    "age": 20
}
```

---

### 리소스 제거2
