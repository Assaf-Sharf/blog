## CPU와 메모리

### CPU는 무엇인가?
- 프로그램의 명령을 실행하고 데이터를 처리하는 역할을 수행

### CPU의 구성요소는?
![](https://teamsparta.notion.site/image/https%3A%2F%2Fs3-us-west-2.amazonaws.com%2Fsecure.notion-static.com%2Fb978c8e0-9005-440c-9da9-c70f9db7809d%2FUntitled.png?id=4b931ed8-6948-41dc-8014-a9beab8d994f&table=block&spaceId=83c75a39-3aba-4ba4-a792-7aefe4b07895&width=670&userId=&cache=v2){: width="100%"}

- ALU(산술논리 연산장치): 연산을 수행
- CU(제어부)와 내부버스: 명령어의 실행 순서를 스케쥴링
- 레지스터: 실행하는 연산에 대한 명령어를 저장
- 캐시 메모리(L1): 연산 처리 속도를 높여주는 메모리

### 레지스터와 L1캐시 메모리의 공통점과 차이점은?
#### 공통점
- 모두 중요한 정보를 저장하고 CPU가 빠르게 액세스 할 수 있도록 한다.
#### 차이점
- 레지스터보다 캐시 메모리가 용량이 더 크지만, 속도는 레지스터가 더 빠르다.
- 레지스터는 현재 처리중인 작업에 관련된 데이터를 저장하지만, 
  L1 캐시 메모리는 CPU가 자주 사용하는 데이터와 명령어를 저장한다.

### L1과 L2, L3 캐시메모리의 공통점과 차이점은?
#### 공통점
- CPU가 자주 사용하는 데이터를 빠르게 액세스하기 위한 저장소
  
#### 차이점
- L1 캐시메모리는 CPU 내부에 존재 데이터나 명령어를 보관
- L2, L3 캐시메모리는 CPU 외부에 존재하고 메인 메모리의 데이터를 캐시 메모리로 불러와 CPU가 필요한 데이터를 캐시에서 먼저 찾도록하여 시스템 성능을 향상시켜줌

### CPU는 어떻게 동작하나?
1. 명령어 인출
    - CU가 수행할 명령어 정보를 가지고옴
2. 명령어 해독
    - opcode라고 하는 명령어 코드를 인출하고 opcode의 성격에 맞게 레지스터들을 준비시킴
3. 실행
    - 해독된 명령어를 수행
4. 반영
    - 명령어의 수행 결과를 반영

### 주기억장치와 보조기억장치의 차이점은?
- 주기억장치는 보조기억장치보다 접근속도가 빠르고, 전원이 꺼지면 메인 메모리에 저장된 내용이 사라지는 휘발성 메모리이다.
- 보조기억장치는 전원이 꺼져도 지워지지 않고 데이터와 프로그램이 반영구적으로 저장되는 비휘발성 메모리이다.

### CPU와 메모리의 구조는 무엇이 있을까?

#### 하버드 구조
![](https://yms218.github.io/assets/images/%ED%95%98%EB%B2%84%EB%93%9C%EA%B5%AC%EC%A1%B0.png){: width="100%"}

- 명령어를 모아둔 명령어 메모리와 명령어들로 사용될 데이터 메모리로 구분
- 메모리가 2개의 역할로 구분되어 있어서 속도가 빠르다는 장점이 존재
- 그러나 구성이 많기 때문에 비싸고, 복잡한 구성으로 고장이 날 확률이 높다

#### 폰노이만 구조
![](https://velog.velcdn.com/images%2Fckstn0777%2Fpost%2F885d1e6f-0260-40db-be45-382e434a277c%2Fimage.png){: width="100%"}

- 메모리를 구분하지 않아 보조기억장치에서 데이터들이 CPU에 연산되기 위해
  하나의 메모리에 올라간다.
- 메모리와 CPU가 하나의 버스(통로)를 가지고 있기 때문에,
  병목현상(처리지연현상)이 발생한다.

### CPU와 메모리는 어떻게 동작하나?
1. **주기억장치**가 **입력받은 데이터**나 **보조기억장치에 저장된 프로그램**을 읽어온다.
2. **CPU**는 **주기억장치**에 저장된 **프로그램 명령어**와 **데이터**를 **읽어와 처리**하고 **결과**를 다시 **주기억장치에 저장**한다.
3. **주기억장치**는 **처리 결과**를 **보조기억장치**에 **저장**하거나 **출력장치**로 보내서 **출력**한다.
4. CPU 내의 **제어장치(CU)**가 1~3번의 과정에서 명령어가 **순서대로 실행**되도록 **각 장치**들을 **제어**한다.
