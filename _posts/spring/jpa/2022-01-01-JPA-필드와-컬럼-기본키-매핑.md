## ✔ 요구사항 정리

- 회원은 일반 회원 & 관리자로 구성
- 회원 가입일 & 수정일이 존재
- 회원을 설명할 필드가 존재 해야한다.
- 해당 필드는 길이 제한이 없다

## ✔ @Column

### name

- 필드와 매핑할 테이블의 컬럼명

### insertable

- 등록 가능 여부 지정 ( INSERT 가능 여부 )

### updatable

- 변경 가능 여부 지정 ( UPDATE 가능 여부 )

### nullable( DDL )

- null값 하용 여부, false 설정시 DDL 생성시 not null 제약 조건 붙는다

### unique( DDL )

- @Table의 uniqueConstraints와 같지만, 한 컬럼에 유니크 제약 조건을 걸 때 사용

### columnDefinition

- 데이터베이스 컬럼정보를 직접 줄 때 사용이된다

### length( DDL )

- 문자 길이에 대한 제약조건 설정

## ✔ @Id

- 기본키 매핑시 사용이 된다

## ✔ @Enumerated

- Java에서 Enum 사용 시 Enumatrated 어노테이션을 사용하면 된다

### Enumerated( EnumType.ORDINAl )

- EnumType.ORDINAL을 운영상에서 사용하면 안된다
- ORDINAL은 ENUM을 숫자( 1, 2, 3 )으로 저장한다
- 즉, EnumType.STRING을 사용

## ✔ @Temporal

- 날짜 설정 시 사용이 되는 어노테이션
- 자바는 날짜, 시간이 다 존재하지만 보통의 DB는 날짜, 시간을 구분하여 사용한다
- Java 8이상에서는 LocalDate, LocalDateTime을 사용해도 Hibernate가 인식한다

## ✔ @Lob

- 매핑하는 필드 타입이 문자면 CLOB 매핑, 나머지 BLOB 매핑
- VARCHAR를 넘어서는 데이터 형식을 표현할 경우 사용

## ✔ 기본 키 매핑 어노테이션

```java
@Id @GeneratedValue(strategy = GenerationType.AUTO)
private Long id;
```

- @Id
- @GeneratedValue

### @GeneratedValue

```java
public enum GenerationType {
    TABLE,
    SEQUENCE,
    IDENTITY,
    AUTO;

    private GenerationType() {

    }
}
```

- 직접 할당: @Id만 사용
- 자동 생성( @GeneratedValue )
  - IDENTITY: 데이터베이스에 위임, MYSQL
  - SEQUENCE: 데이터베이스 시퀸스 오브젝트 사용, ORACLE
    - @SequenceGenerator 필요
- TABLE: 키 생성용 테이블 사용, 모든 DB에서 사용
  - @TableGenerator 필요
- AUTO: 방언에 따라 자동 지정, 기본값

## ✔ 직접 할당

- @Id 사용

### IDENTITY 전략

```java
@Entity
public class Member {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
}
```

```java
Hibernate:
    /* insert hellojpa.Member
        */ insert
        into
            Member
            (id, age, createdDate, description, lastModifiedDate, roleType, name)
        values
            (null, ?, ?, ?, ?, ?, ?)
```

- 기본 키 생성 권한을 DB에 위임
- 즉, DB가 기본 키 생성에 대한 모든 것을 관리하도록 제어
- JPA는 보통 트랜잭션 커밋 시점에 ISNERT SQL을 실행한다는 특징이 있다
- 즉, DB에 INSERT가 된 후에 시퀸스가 생성이 된다

### SEQUENCE 전략

```java
@Entity
@SequenceGenerator(
        name = "MEMBER_SEQ_GENERATOR",
        sequenceName= "MEMBER_SEQ", //매핑할 데이터베이스 시퀀스 이름
        initialValue= 1, allocationSize = 50)
public class Member {
    @Id
    @GeneratedValue(strategy = GenerationType.SEQUENCE)
    private Long id;
}
```

- 데이터베이스에 있는 시퀸스 값을 가져와서 넣어준다
- 데이터베이스 시퀸스는 유일한 값을 순서대로 생성하는 특별한 DB Object
- Oracle, PostgreSQL, DB2

### @SequenceGenerator Options

- name
  - 식별자 생성기 이름
- sequenceName
  - DB에 등록된 시퀸스 이름
- initialValue
  - DDL 생성시 사용, 최초 sequence값 결정
- allocationSize
  - 시퀸스 한번 호출 시 증가값 설정
  - call next value for MEMBER_SEQ 호출
    - DB에 50개를 미리 올리고 메모리에서 1씩 사용
  - 첫 번째는 DB에서 seq값을 가져온다
    - 두 번째부터는 DB가 아닌 memory에서 값을 가져온다
- catalog, schema
  - DB catalog, schema 이름

### TABLE 전략

```java
@Entity
@TableGenerator(
    name = "MEMBER_SEQ_GENERATOR",
    table = "MY_SEQUENCES",
    pkColumnValue = "MEMBER_SEQ", allocationSize = 1)
public class Member{
    @Id
    @GeneratedValue(strategy = GenerationType.TABLE,
            generator = "MEMBER_SEQ_GENERATOR")
    private Long id;
}
```

- 키 생성 전용 테이블 생성 후 시퀸스 관리
- 장점: 모든 DB에 적용 가능
- 단점: 성능 저하

### @TableGenerator Options

- name
  - 식별자 생성기 이름
- table
  - 키 생성 테이블명
- pkColumnValue
  - 시퀸스 컬럼명
- valueColumnNa
  - 시퀸스 값 컬럼명
- pkColumnValue
  - 키로 사용할 값 이름
- initialValue
  - 초기 값, 마지막으로 생성된 값이 기준
- allocationSize
  - 시퀸스 한 번 호출에 증가하는 수
- Catalog
  - DB catalog, schema 이름
- uniqueConstraints(DDL)
  - 유니크 제약 조건 지정

### 권장하는 식별자 전략

- 기본 키 제약 조건
- 미래까지 이러한 조건을 유지하는 자연키 찾기 어려움
- PK값은 type을 Long형 + 대체키 + 키 생성전략
