## ✔ 양방향 매핑시 주의점

```java
// Class : MemberTest
@Entity
@Table(name = "MEMBER_TEST")
public class MemberTest {

  @Id
  @GeneratedValue
  @Column(name = "MEMBER_ID")
  private Long id;

  @Column(name = "USERNAME")
  private String username;

  @ManyToOne
  @JoinColumn(name = "TEAM_ID")
  private TeamTest team;
}

// Class : TeamTest
@Entity
@Table(name = "TEAM_TEST")
public class TeamTest {

  @Id
  @GeneratedValue
  @Column(name = "TEAM_ID")
  private Long id;
  private String name;

  @OneToMany(mappedBy = "team")
  private List<MemberTest> members = new ArrayList<MemberTest>();

  public List<MemberTest> getMembers() {
    return members;
  }
}
```

- 위 코드를 보면 현재 연관관계의 주인은 MemberTest 클래스의 TeamTest 객체
- 연관관계의 주인에 값을 입력하지 않는다
- 양방향 매핑시에는 양쪽에 값을 넣어주는게 맞다
- 순수 객체 상태를 고려해서 항상 양쪽에 값을 설정한다
- 연관관계의 편의 메서드를 생성한다
- 양방향 매핑시에 무한 루프를 조심한다
  - toString(), lombok, JSON 생성 라이브러리
- 양방향 연관관계 매핑을 할 경우 연관관계의 주인에 따라 값을 넣어준다
- 양쪽에 넣어주거나 주인쪽에 값을 넣어줘야 한다

### 양방향 매핑시 연관관계의 주인에 값을 입력

```java
TeamTest team = new TeamTest();
team.setName("팀2");
em.persist(team);

MemberTest member = new MemberTest();
member.setUsername("사용자2");

team.getMembers().add(member);
//연관관계의 주인에 값 설정

member.setTeam(team) //**

em.persist(member);
```

### 양방향 매핑 정리

- 단방향 매핑만으로도 이미 연관관계 매핑은 완료
- 양방향 매핑은 방대 방향으로 조회( 객체 그래프 탐색 ) 기능이 추가된 것 뿐
- JPQL에서 양방향으로 탐색할 일이 많음
- 단방향 매핑을 잘 하고 양방향은 필요할 때 추가해도 됨 ( 테이블에 영향을 주지 않음 )
- 즉, 양방향 매핑은 지양하고 단방향 매핑으로 전부 끝낸다는 생각으로 진행 해야 한다

### 연관관계의 주인을 정하는 기준

- 비즈니스 로직을 기준으로 연관관계의 주인을 선택하면 안된다
- 연관관계의 주인은 외래 키의 위치를 기준으로 정해야 한다
