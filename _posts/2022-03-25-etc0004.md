<h3>[자바(Java) 공부전에 알아두면 좋을 것들 ]</h3><br>

<h6>기초 용어</h6>

1. 컴파일 : 프로그래밍 언어를 컴퓨터가 이해할  수 있는 바이트코드로 변환 <br>
2. 컴파일러 : 컴파일을 해주는 프로그램으로 통상 운영체제별 다른 컴파일러가 필요한데 <br>자바는 Java 설치시 JVM(자바 가상머신:java virtual machine)이 같이 설치되며 <br>중간에서 운영체제에 맞게 번역을 한다.<br>
3. JRE(Java Run Environment): JVM + 자바프로그램 실행에 필요한 여러 라이브러리 파일<br> ex) print, scanner 등<br>
4. JDK(Java Development Kit) JRE + 개발을 위한 도구를 가지고 있다. ex) 컴파일러, 디버그<br>JDK 설치 시 JRE와 JVM이 동시에 설치되기에 통상 자바의 버전을 얘기할때는 JDK의 버전을 말하는 것이다.
5. JDK 버전은 업데이트 될 때마다 기능이 추가되고 일부 LTS(Long Time Support)라고 써 있는 버전은<br> 오랫동안 지원해주는 버전으로 주로 최신 버전 중 LTS 버전을 선택하는게 좋다.<br>
  또한 JDK는 주로 DB 종류별로 다양한 버전이 있다.

   빌드 소스코드 파일을 컴퓨터에서 실행할  수 있는 독립 SW 가공물로 변환시키는 과정
   독립 SW 가공물 = Artfact
   빌드를 세분화 하면
7. 소스코드를 컴파일한다.
8. 테스트코드를 컴파일한다.
9. 테스트코드를 실행한다.
10. 테스트코드 리포트를 작성한다.
11. 기타 추가 설정한 작업들을 진행한다.
12. 패키징을 수행한다.
13. 최종 소프트웨어 결과물(Artifact)을 만들어낸다.
