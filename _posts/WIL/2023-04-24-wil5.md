### ORM이란?<br>


- Object Relational Mapping (객체-관계-매핑) 의 약자로,     프로그래밍 언어의 객체와 관계형 데이터베이스의 데이터를 자동으로 연결해주는 도구이다. 

  객체지향 프로그래밍에서는 객체와 클래스를 사용하고, 관계형 데이터베이스는 데이블을 사용하기 때문에 객체 모델과 관계형 모델간의 불일치가 존재하는데, ORM을 통해 객체 간의 관계를 바탕으로 SQL을 자동으로 생성하여 불일치를 해결할 수 있다.

  이를 통해 개발자는 객체 지향 프로그래밍 언어로 데이터베이스를 간접적으로 다룰 수 있게 되며, 코드의 가독성과 유지보수성을 향상시킬 수 있다.


### SQL vs noSQL


- SQL은 Structured Query Language의 약어로, 데이터를 테이블 형태로 저장하고 관리하는 관계형 데이터베이스(RDBMS)를 의미한다.<br> 
  SQL을 사용하면 테이블 간의 관계를 정의하고 쿼리를 이용하여 데이터를 검색, 추가, 수정, 삭제할 수 있다.<br>
  명확하게 정의 된 스키마를 가지고 있어 데이터의 일관성과 정확성을 보장하고 관계를 통해 각 데이터를 중복없이 한 번만 저장할 수 있다.


- noSQL은 Not Only SQL의 약어로, 관계형 데이터베이스와는 달리 테이블 형태가 아닌 다양한 데이터 모델을 사용하여 데이터를 저장하고 관리하는 비관계형 데이터베이스를 의미한다.<br>
  스키마가 없기때문에, 저장된 데이터를 언제든지 조정하고 새로운 필드를 추가할 수 있다.<br> 
  수직 및 수평 확장이 가능하므로 데이터베이스가 애플리케이션에서 발생시키는 모든 읽기/쓰기 요청을 처리 할 수 있다.
