## 1.  &nbsp;전 블로그에서 작성한 것들📝  

<br/> 


- __ls__로 현재 위치 확인 
- __mkdir + 폴더이름__ 폴더 생성
- __cd + 폴더이름__으로 해당 폴더로 위치한다
- __touch + 파일이름__ 파일 생성
- __rm + 파일이름__ 파일 제거
- __rm -rf + 폴더이름__ 폴더 제거
- __vi + 파일이름__ vim으로 해당 파일 열기
- __cat + 파일이름__ 파일 미리보기
- __git status__ 변경사항 확인
- __git add + 파일이름__ add하기
- __git commit__ commit하기
- __git push (-u) origin + branch이름__ push하기

#### Github<br/> 

- github에서 repository를 만들고 주소를 복사한다
- Vscode 내부 terminal에서 cd 명령어로 작업 폴더에 위치한다
- __git clone + 주소__ 입력해서 해당 repository 폴더를 가져온다
- cd 명령어로 그 폴더에 들어간다 (__main__ branch가 생성되어있다)
- 작업을 한다
- __git add + 파일이름__ 입력해서 __add__를 해준다
- __git commit__ 입력해서 commit을 작성해준다
- __git push origin main__ 입력해서 github에 __push__해준다

#### ⭐️팀원들과 프로젝트 협업할 때<br/> 

1. 권한을 부여

   - 팀장 레포지토리 => 설정 => manage access => add people    __문서 작업__을 맡길 때 정도에만 사용<br/> 

2. __⭐️git flow__

   - __👀팀장__

     - repository 생성 => Clone => git flow init => 간단한 작업(기본 파일 생성 + .gitignore 생성) <br/>=> 기본 파일들과 .gitignore 파일 구분해서 __add__한다 (feat | conf) => commit => git push __-u__ origin develop<br/> 

   - __🧑🏻‍💻팀원__

     - 팀장 레포 주소 복사 => 팀장 레포에 이슈를 작성한다 (뭘할지 제목 + checkbox) => __fork__하면 <br/>=>  내 레포에 팀장 레포 복사됨 => vscode 가서 => __git clone + 내 복사된 레포 주소__ <br/>=>__git flow init__해서 develop branch 가져오기 (안에 팀장이 생성한 파일들 있음)
     - __git flow feature start + 이름__ 입력해서 __feature branch__를 생성한다 => 작업하고 __add | commit__ <br/>=> commit할 때마다 __이슈 checkbox__ 체크하면서 한다.
     - 작업이 __완전 완료__되면 commit에서 작성할 때 __fix | close | resolve + #이슈 넘버__해서 작성 <br/>=> __git flow feature finish + 이름__ 입력해서 develop branch로 __자동 merge__한다
     - __git push origin develop__ 입력해서 내 레포에 작업한 develop branch __push__한다
     - __compare & pull request__해서 팀장님께 보고
     - 보고 내용에 __fix | close | resolve + #이슈넘버__ 작성
     - 🛠 수정사항이 필요하면 develop branch에서 바로 수정하고 push해도 됨 (__refactor__)<br/> 

   - __📎 merge된 팀장님 최신 레포 가져오기__

     - 👀팀장

       - git fetch origin develop
       - git merge FETCH_HEAD <br/> 

     - 🧑🏻‍💻팀원

       - __git remote__하면 origin(__내 origin__)이 보임

       - __git remote -v__ 입력해서 팀장님 upstream이 없으면

       - __⭐️팀장님 레포 주소 복사__해서 __git remote add upstream + 주소__ (upstream에 주소 대입)

       - __git fetch upstream develop__

       - __git merge FETCH_HEAD__

#### 한글 난독화 사이트 클론 <br/> 

🖥HTML부분을 맡았다



```html
<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <meta http-equiv="X-UA-Compatible" content="IE=edge">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>AIRBNB 난독화 연습</title>
    <!-- reset css cdn -->
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/reset-css@5.0.1/reset.min.css">
    <!-- css -->
    <link rel="stylesheet" href="./main.css">
    <!-- JS -->
    <script defer src="./main.js"></script>
</head>
<body>
  <!-- 한글 난독화 -->
  <header>
    <h1>
      <a href="/index.html">한글 난독화</a>
    </h1>
  </header>
  <!-- 메인 -->
  <section>
    <!-- 변환되기 전 -->
    <article class class="before">
      <textarea class="box" placeholder="변환할 문장들을 여기에 입력하고 '변환하기' 버튼을 누르세요."></textarea> 
      <!-- 버튼 -->
      <div class="btn-area">
        <button class="btn btn--primary">변환하기</button> 
        <button class="btn btn--assist">입력창 지우기</button>
        <button class="btn btn--assist">변환 옵션</button>
      </div>
      <!-- 변환 옵션 -->
      <div class="option-area">
        <form>
          <div class="option">
            <label class="option--text" for="option1">소리나는 대로 연음법칙 적용</label>
            <input type="range" id="option1">
          </div>
          <div class="option">
            <label class="option--text" for="option2">뒤에 오는 자음을 받침으로 중복</label>
            <input type="range" id="option2">
          </div>
          <div class="option">
            <label class="option--text" for="option3">자모를 비슷한 발음으로 변환</label>
            <input type="range" id="option3">
          </div>
          <div class="option">
            <label class="option--text" for="option4">의미없는 받침 추가</label>
            <input type="range" id="option4">
          </div>
          <div class="check">
            <input id="checkBox" type="checkbox">
            <label for="checkBox">완성형 글자만 허용</label>
          </div>
        </form>
      </div>
    </article>
    <!-- 변환된 후 -->
    <article class="after">
      <h4>변환된 문장:</h4>
      <textarea class="box" placeholder="변환된 문장들이 여기에 나타납니다."></textarea>
      <button class="btn btn--assist">클립보드에 복사</button>
      <p>
        한국인만 읽을 수 있는
        <a href="javascript:void(0)">한자문장으로 바꾸려면 여기</a>
      </p>
    </article>
  </section>
</body>
</html>
```

- 적응형 vs 반응형

  \- 반응형 : 컨텐츠의 배치 정도만 바뀔 가벼운 서비스를 웹 사이즈 변경시에 따라 구조를 바꿈
- 쿠키 : 사용자의 브라우저(컴퓨터)에 저장 (안전한 정보만 저장)
- 세션 : 서버에 저장
- 캐시 : 용량이 큰 데이터를 다시 다운받을 필요없이 재사용을 목적으로 하는 것
- API : 프로그램이 서로 소통하는 방법? ex. 날씨 API
- 디버깅 : 오류제거
- 컴파일 : 무거운언어 (자바, C etc..)를 기계가 알아들을 수 있게 변환
- 빌드 : 사용자가 사용할 수 있게 최종 포장 단계
- 프레임워크 : React, 전체 틀?
- 라이브러리 : 특정 기능을 구현하는 코드
- 호스팅 : 서버를 빌려주는 원룸?
- 클라우드 : 서버를 빌려주는 호텔? <br/>
