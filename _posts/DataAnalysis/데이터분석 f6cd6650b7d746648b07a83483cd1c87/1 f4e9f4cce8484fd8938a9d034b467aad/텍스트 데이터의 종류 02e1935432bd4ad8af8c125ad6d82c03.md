# 텍스트 데이터의 종류

- **CSV/TSV**
    
    <aside>
    💡 구분 텍스트
    
    데이터의 줄바꿈으로 행을, 구분자로 열을 구문하는 텍스트 데이터를 말한다.
    구분자는 공백, 세미콜론, 콜론, 슬래시 어떤 것이라도 가능하도. 보통 쉼표와 탭 구분자가 가장 일반적이다.
    
    </aside>
    
    CSV와 TSV는 웹에서 굉장히 많이 사용된다. 그 이유는 구조가 굉장히 단순하고 엑셀로도 쉽게 만들수가 있으며, 수많은 데이터베이스와 데이터도구 등에서 CSV 혈싱을 지원하고 있기 때문이다.
    
    CSV와 TSV의 가장 큰 차이점은 데이터 추출 후 분석을 하던 XML, JSON, YAML와 달리 별도의 데이터 추출 과정 없이, 데이터 변환없이 바로 분석활용이 가능하다.
    
    - CSV(Comma-Sparated Values)파일은 각 필드를 쉼표로 구분한다.기본적으로 텍스트 파일이므로 텍스트 에디터를 사용해 간편하게 수정할 수 있다.
    - TSV(Tab-Sparated Values)파일은 CSV와 비슷하지만 쉼표가 아닌 탭으로 필드를 구분한다.
    - SSV(Space-Sparated Values)파일은 공백으로 필드 구분한다.
    
    **CSV파일의 구조** 
    
    - 1개 이상의 record로 구성되며, record는 줄바꿈(CRLF, U+00D와 U+000A)으로 구분된다.
    - CR → 줄바꿈 → \r
    - LF → 커서이동 → \n
    - 실습 : 파이썬에서 csv, tsv 파일 읽기
        
        단순한 데이터를 다룰 때는 줄바꿈과 쉼표로만 구분하면 된다. 따라서 별도의 모듈을 사용할 필요가 없다. 문자 코드 인코딩만 주의하면 된다.

- **YAML**
    
    <aside>
    💡 YAML
    들여쓰기를 사용해 계층 구조를 표현하는 것이 특징인 데이터 형식이다. 텍스트 데이터이므로 텍스트 에디터를 사용해 편집할 수 있다.
    
    </aside>
    
    - YAML은 공백문자로 들여쓰기를 활용해 계층구조를 나타내는 점에서 파이썬과 비슷하다. 전체적인 데이터 구조를 표현한다.
    - 단순화 : YAML > JSON > XML
    - JSON 대용으로도 사용되며 application 설정 파일을 작성 할 때 많이 사용한다. 루비/ 파이썬/ PHP 등을 포함한 여러 프로그래밍 언어에는 YAML형식을 다루기 위한 라이브러리가 제공된다.
    - **YAML 데이터 형식**
        
        YAML의 기본은 배열(list)과 해시(dict), 스칼라(문자열, 숫자, 불리언 등) 이다.
        
        - 배열을 나타낼 때는 각 행의 앞에 하이픈(-)을 붙인다. 하이픈(-) 뒤에는 공백이 필요하다 .
        - 해시를 나타낼 때는 JavaScript의 객체와 같다.<key> : <Value> 형태로 사용된다.
        - 들여쓰기 이용 시 계층 구조 표현 할 수 있다.
        - 배열과 해시를 조합하면 복잡한 데이터를 표현 할 수 있다.

- **XML**
    
    <aside>
    💡 **XML,** Extensible Markup Language
    XML은 텍스트 데이터를 기반으로 하는 형식이다. 주로 다른 시스템끼리 데이터를 쉽게 주고 받을 수 있도록 HTML의 한계를 극복할 목적으로 만들어졌다. 많은 웹API가 XML 형식을 활용한다.
    
    </aside>
    
    원래 XML(Extensible Markup Language)는 특정 목적에 따라 데이터를 태그로 감싸 마크업하는 범용적인 형식이다.
    
    XML은 데이터를 계층구조로 표현 할 수 있는 것이 장점이다. 이러한 계층 구조는 태그로 표현한다.
    
    ```
    <요소 속성 = "속성값">내용</내용>
    ```
    
    원하는 데이터 <요소>태그로 감싸 마크업한다. 하나의 요소에는 속성을 사용하여 여러값을 추가로 지정할 수 있다.
    
    - sitemap.xml
        - sitemap.xml은 웹사이트 내 모든 페이지의 목록을 나열한 파일로 책의 목차와 같은 역할을 한다. 사이트맵을 제출하면 일반적인 크롤링 과정에서 쉽게 발견되지 않는 웹페이지도 문제없이 크롤링되고 색인될 수 있게 해준다.
            
            [](https://wordpress.com/sitemap-1.xml)
