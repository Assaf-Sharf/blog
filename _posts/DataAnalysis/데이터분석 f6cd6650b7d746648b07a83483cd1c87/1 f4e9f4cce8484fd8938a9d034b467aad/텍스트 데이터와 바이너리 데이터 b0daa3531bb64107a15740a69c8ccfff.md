# 텍스트 데이터와 바이너리 데이터

XML/JSON/YAML/CSV/TSV/EXCEL/PDF 등 웹에는 다양한 데이터가 있다.

데이터 포맷은 텍스트 데이터와 바이너리 데이터로 구분한다.

- **텍스트 데이터**
    
    <aside>
    💡 문자 데이터로 사용할 수 있는 영역. 일반적으로 텍스트 에디터로 편집할 수 있는 데이터 포맷. 
    일반적인 자연언어(한국어, 영어, 한자 등)와 숫자 등으로 구성된다. 
    특수하게 줄바꿈과 탭등의 제어 문자도 포함되어 있다. 
    이외에는 모두 에디터에서 시각적으로 확인할 수 있는 데이터 형식이다.
    
    </aside>
    
    [장점] 텍스트 에디터가 있으면 편집할 수 있다. 설명 포함도 가능하여 가독성이 높다.
    [단점] 바이너리 데이터에 비해 크기가 크다.
    
    - 프로그래밍 언어의 소스코드도 텍스트 데이터. ML/JSON/YAML/CSV…웹에서 주로 사용되는 데이터 포맷은 텍스트 데이터를 기반으로 한다.
    - 최근의 텍스트 데이터를 기반으로 하는 곳은 모두 utf-8로 인코딩한다. HTML에서 utf-8사용을 권장하기 때문이다. (문자 코드 인코딩 시 주의해야 한다.)
    - 실습
        
        텍스트 데이터는 문자가 적혀있는 데이터이다. 어떤 문자 코드(문자 인코딩)으로 저장돼 있냐에 따라 다른 의미를 갖게 된다. 같은 문장의 텍스트 파일이라도 문자 인코딩이 다르면 다른 문자를 표현한다.
        
        그 예로, euc_kr(줄바꿈 CRLF)와 utf-8(줄바꿈 LF)로 나타낼 때 같은 문장의 텍스트 데이터라도 파일을 열어봤을 때 문자가 깨져있을 수 있다.

- **바이너리 데이터**
    
    ```
    텍스트 데이터 이외의 데이터. 
    문자와 상관없이 데이터를 사용할 수 있는 데이터 영역을 활용하는 데이터 형식이다. 
    바이너리 데이터는 문자에 할당되는 영역 외의 영역도 사용하므로 일반적인 텍스트 에디터로는 열 수 없다.
    ```
    
    [장점] 텍스트 데이터에 비해 크기가 작다.
    [단점] 텍스트 에디터로 편집할 수 없다. 어떤 바이트에 어떤 데이터가 있다고 정의해야한다.
    
    웹에서 사용되는 데이터는 대부분 **바이너리 데이터**이다. 그 이유는 이미지와 동영상 같은 파일 때문이다. 이미지와 동영상 파일은 용량이 크므로 최대한 서버의 부하를 줄이기 위해 크기를 압축해야 하기 때문이다. 이미지와 동영상의 경우에는 이미지에 따라 JPEG/GIF/PNG 등의 형식을 이용한다.
