## Librería y Framework

* Son herramientas de **desarrollo** que se utilizan para agilizar el trabajo en la codificación de un sistema concreto

* Dependiendo del objetivo que queramos alcanzar se utiliza una herramienta u otra

## Concepto Framework

* Se usa para desarrollar un proyecto concreto con una estructura definida y donde la compatibilidad de sus funcionalidades este asegurada

* Los **framework** te dan una **estructura completa** para desarrollar un proyecto profesional desde el principio hasta el final añadiendo funcionalidades sin usar elementos externos como **librerías de terceros**

* Tiene asegurada la compatibilidad entre sus componentes y elementos los cuales te dan unas pautas de trabajo muy bien definidas sobre como generar tu proyecto

* Ofrece una **estructura básica** para el desarrollo completo de tu proyecto con una integración sin necesidad de depender de librerías de terceros

### Uso de Framework

* Se utiliza para crear pequeños , medianos y grandes proyectos sin tener que usar elementos de terceros y siguiendo unas pautas de desarrollo muy bien definidas

 * Si necesitas un validador de formularios **(el framework tiene incorporada la validación de los formularios)**

 * Si necesitas imprimir datos en pantalla **(el framework trae la forma de imprimir esos datos por pantalla u otro dispositivo que le indiquemos)**


### Ejemplos de Framework

* **Angular** : utilizado para el **desarrollo frontend** mediante el **patrón de diseño MVC** ``(Modelo Vista Controlador)`` y esta pensando para realizar ``SPAs (Single Page Applications)`` desarrollo web en una sola pagina

* **Vue** : centrado sobre todo en la capa de visualización
 
### Lo bueno de las Framework

* Te da un entorno o un marco de trabajo seguro , estable y bien definido para que tu proyecto gane en productividad

* Tienes todas las **soluciones o funcionalidades empaquetadas** listas para añadirlas a tu proyecto según vayas necesitando las cuales te indican que hacer , como hacerlo y como mandarlo a producción una vez terminadas
 
### Lo malo de las Framework

* Estamos limitados a las exigencias y patrones de desarrollo del **framework**

* No ofrece tanta libertad como las **librerías** , tiene flexibilidad limitada
  * No puedes salirte de las pautas de trabajo marcadas

* Desarrollos muy homogéneos , estandardizado y lineales **(aunque escalables)**
