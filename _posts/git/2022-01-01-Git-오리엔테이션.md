## ✔ Class Introduction

### Git?

- **Git = Version Control System (버전 관리 시스템)**

### Version Control?

- 파일의 이름을 더럽히지 않는 버전 관리를 할 수 있다?
- 파일의 버전 관리에 대한 권한을 위임
- report.xls -> report*final.xls -> report*백업.xls

### Version Control?

**버전 관리란 무엇이고, 우리는 왜 이것을 알아야 하는가?**

- **파일의 변화**를 **시간**에 따라 기록했다가 특정 시점의 버전을 다시 꺼내올 수 있는 시스템
- **파일이 손실** 되거나 **잘못 고쳤을 경우**, 이전 상태를 다 저장해 두기에 복구가 가능하다

### 로컬 버전 관리

<img width="450" alt="스크린샷 2021-07-03 오후 6 26 08" src="https://user-images.githubusercontent.com/53969142/124349831-39b88d80-dc2c-11eb-934c-5b6e76e1a0b8.png">

- 많은 사람들은 버전 관리를 위해 디렉토리 파일을 복사
  - 위 방식은 위험성이 높다
- 이런 이유로 인해 로컬 `VCS` 탄생
  - **아주 간단한 데이터베이스를 이용하여 파일의 변경 정보 관리**

### 중앙 집중식 버전 관리

<img width="450" alt="스크린샷 2021-07-03 오후 6 29 19" src="https://user-images.githubusercontent.com/53969142/124349923-9ae06100-dc2c-11eb-8136-168d7daa9180.png">

- 프로젝트 진행 시 다른 개발자와 함께 작업해야 하는 경우 존재
- **중앙 서버** 한 대를 두고, **클라이언트** 측에서는 중앙 서버의 파일을 받아 사용
- **중앙 서버에 문제 발생 시, 복구 힘듬**

### 분산 버전 관리 시스템

- 대표적으로는 깃이 분산 버전 관리 시스템에 해당
- 로컬에 저장소를 하나 더 만들어 add, commit, push 명령어를 사용해 중앙 서버에 업로드
- 버전 관리에 효율적이며 중앙 서버에 문제가 생겨도 복구 가능
- 파일 손실 및 유실에도 이전 버전을 통해 해당 파일 복구 가능
- 오픈소스, 전 세계의 엄청난 개발자들의 코드를 마음껏 볼 수 있는 커뮤니티

### 버전 관리의 특징

- **Backup(백업)**
- **Recovery(복구)**
- **Collaboration(협업)**
