## 1. 설치

[Tipue Search](https://github.com/jekylltools/jekyll-tipue-search) 설치하자.

![image](https://user-images.githubusercontent.com/46878973/170419420-9cef9350-645d-47c0-903b-9848778631a0.png)

설치 후 해당 폴더내의 search.html 파일을 본인의 깃허브 블로그 최상위 디렉토리에 복사한다.  

assets 폴더안의 tipuesearch 폴더를 본인의 깃허브 블로그 최상위 디렉토리안의 assets 폴더안에 복사한다.

## 2. 환경설정

### 2.1. _config.yml 수정

본인의 깃허브 블로그 최상위 디렉토리 안의 _config.yml에 아래 코드를 추가한다

```
tipue_search:
     include:
         pages: false
         collections: []
     exclude:
         files: [search.html, index.html, tags.html]
         categories: []
         tags: []
```

- include > pages: false의 설정은 pages 레이아웃에 해당하는 일반 html페이지는 검색하지 않겠다는 것을 의미한다.(포스트 내용 검색에 집중하기 위함)
- exclude > search.html, index.html, tags.html 페이지는 검색에서 제외하겠다는 것을 의미한다.

### 2.2. head.html 수정

본인의 깃허브 블로그 최상위 디렉토리 > _includes > head.html 에 아래 코드를 추가한다.  

맨 아래 meta와 link 바로위 사이에 추가하면 된다.

![image](https://user-images.githubusercontent.com/46878973/170424996-808de958-af8f-44c9-977e-245294e7c3c3.png)

```html
 <!-- tipuesearch -->
 <link rel="stylesheet" href="/assets/tipuesearch/css/tipuesearch.css">
 <script src="https://ajax.googleapis.com/ajax/libs/jquery/3.2.1/jquery.min.js"></script>
 <script src="/assets/tipuesearch/tipuesearch_content.js"></script>
 <script src="/assets/tipuesearch/tipuesearch_set.js"></script>
 <script src="/assets/tipuesearch/tipuesearch.min.js"></script>
 ```

### 2.3. search.html 수정

본인의 깃허브 블로그 최상위 디렉토리의 search.html에 코드를 추가한다.

![image](https://user-images.githubusercontent.com/46878973/170425569-375fd6af-8f55-493b-9886-a57c19e0b8c2.png)

- layout : page 부분은 포스팅이 담기는 레이아웃 명칭이다.(테마에 따라 다를 수 있음)

- permalink: /search/ 부분은 다음 단계에서 설정할 검색어 및 버튼 Element의 form 태그 내 action 속성과 일치시켜야 한다.

- 'wholeWords' : false 속성은 한글 검색을 가능하게 하는 옵션이다.

- 'showTime' : false 속성은 검색이 완료되기 까지 소요된 시간을 표시하는 옵션이다.

- 'minimumLength' : 1 속성은 최소 검색 글자수에 대한 설정으로 일단은 한단어 이상이면 검색가능하게 설정하였다.  
그 외의 옵션은 Tipue 메인홈페이지 Tipue Search에 접속하여 Options in the .tipuesearch() method에서 상세하게 확인할 수 있다.

```
    'wholeWords' : false,
    'showTime' : false,
    'miniumLength' : 1,
```



Configure or install 버튼을 누르면 설치할 저장소가 나온다. 

![image](https://user-images.githubusercontent.com/46878973/163756508-a7ad18d1-c272-4815-980f-c1b8ed7787ec.png)

댓글을 이슈로 관리할 저장소를 선택한다.

## 2. 설정

설치 완료 후 설정 페이지에서 저장소를 설정해준다.

![image](https://user-images.githubusercontent.com/46878973/163758844-62518077-bd7c-4056-b08a-3dbefd212950.png)  

그 후 블로그 포스트와 이슈 매핑 방법에 대해 설정한다.

![image](https://user-images.githubusercontent.com/46878973/163757625-d7f4aa04-9249-443d-93c4-be27c8475fd9.png)

- 블로그 글 경로를 이슈의 제목으로 설정했다.  
글의 제목은 자주 수정될 수 있지만, 파일명은 수정하지 않을것으로 판단했기 떄문에!

이슈 라벨과 테마 설정을 한다.  
해당 Enable Utterances는 복사 후 _config.yml에 입력 적용한다.
- minimal-mistakes를 사용시 _config.yml에 적용
- 그외 사람들은 _layout/post.html에 추가하면 된다.   

![image](https://user-images.githubusercontent.com/46878973/163759375-657fdfa4-ba57-45f3-a431-62e75bea96b9.png)


repository, comments > provider, utterances > theme, issue_term 이 4가지를 변경해주면 된다.

![image](https://user-images.githubusercontent.com/46878973/163759322-bfd16014-ba97-4748-98a2-4a865c1e68dd.png)

- repository
  - 댓글 Issue 가 올라올 곳으로 선택한  저장소의 permalink

- comments > provider
  - utterances 를 앞으로 사용할 것이므로 utterances 를 입력

- utterances > Theme
  - comments의 테마설정

- utterances > issue_term
  - 설정한 맵핑 키 (pathname)
