## 1. 데이터 엔지니어링에 대해

데이터 산업이 전례없는 속도로 성장을 거듭하고 있다. 산업, 기관, 정부에 걸쳐서 다양한 분야에서 데이터를 사용하고 있다. 하지만 아무 데이터나 막 쓸 수는 없는 법. 수 많은 데이터에서 가치있는 정보를 추출하기 위해서는 두 가지의 중요한 핵심가치를 지켜야 한다. **데이터 정확성**과 **데이터 접근성**이 바로 그것이다. 데이터가 정확하지 않으면 데이터의 신뢰성이 떨어져 원하는 정보를 제대로 도출해내지 못할 것이고, 좋은 데이터가 있다 하더라도 그 데이터에 빠르고 정확하게 접근하지 못한다면 또한 데이터의 가치는 급락할 것이다.  

바로 이러한 점에서 데이터 엔지니어가 필요하다. 아무리 좋은 데이터 분석도구가 있어도 좋은 데이터가 없다면 말짱 도루묵일 것이다. 데이터 엔지니어는 이러한 점에서 데이터 과학의 토대를 받치고 있다고 해도 과언이 아니다. 실제로 데이터 엔지니어링은 데이터 분석의 대부분을 차지하고 있다.
<img src="https://drive.google.com/uc?export=view&id=1oRnqfiTWskFlZqiM37q8JqRCK7khhX07">

이 포스트 시리즈는 데이터 엔지니어링에 대한 전반적인 정리를 진행할 예정이다. 데이터 엔지니어에 대해 관심이 굉장히 많지만 데이터 분석에 비해 얻을 수 있는 자료가 희박하다는 것을 깨닫고 여기저기서 정보를 수집하여 포스트로 정리할 것이다. 많은 부분 coursera의 데이터 엔지니어링 강의를 기반으로 하였고, 필요한 부분이 있으면 여기저기서 정보를 수집하여 정리했다.

## 2. 현대의 데이터 생태계

늘어나는 데이터 처리 속도와 용량, 새로운 빅데이터 툴의 등장, 새로운 데이터 소비자의 지속적인 등장은 데이터가 앞으로도 꾸준히 데이터 분야가 성장할 것이라고 말해주고 있다. 빠르게 바뀌는 데이터 환경 속에서 우리는 몇 가지 알아보고 가야할 것들이 있다.

먼저 현대 데이터 생태계에 대해 살펴보고 가보자. 현대 데이터 생태계의 특징은 **상호연결성**, **독립성**, **지속가능한 성장**이다. 보통의 전사적 데이터 환경에는 분산된 소스로부터의 데이터 통합, 인사이트를 위한 다양한 분석기법, 새로운 인사이트를 활용할 이해관계자, 필요한 데이터를 저장, 처리하기 위한 툴, 어플리케이션, 인프라 등이 있다.  

먼저 데이터 소스에 대해 간단하게 알아보자. 데이터 소스는 크게 정형 데이터와 비정형 데이터로 나눌 수 있다. 이 안에는 텍스트, 이미지, 음성, 클릭, 소셜 미디어, IoT, 데이터베이스 등등 아주 다양한 종류가 있다. 이 데이터로 작업하기 위해서 가장 첫번째로 해야하는 것은 원데이터를 카피하여 저장소에 저장하는 것이다. 이 과정에서 직면하는 과제는 데이터의 신뢰성, 보안성, 통합성이 있다.  

이렇게 가져온 Raw 데이터는 정제되고 접근을 위한 최적화가 필요하다. 이 과정에서의 핵심과제는 높은 접근성과 유연성, 보안성을 지닌 저장소 관리이다.

이렇게 저장된 데이터를 이제 누군가가 사용해야 된다. 사용자에는 비즈니스 이해관계자, 앱, 개발자 및 분석가 등이 있다. 여기서의 과제는 최종 사용자들이 잘 사용할 수 있도록 도와주는 인터페이스, API, 어플리케이션을 구축하는 것이다.

현재 데이터 환경에서 가장 떠오르는 기술은 이제는 데이터 사이언스를 위해서라면 너무나도 필수적으로 알아야할 요소들이 되었다. 여기에는 크게 3가지가 있다.

>1. 클라우드 컴퓨팅
>2. 머신러닝
>3. 빅데이터

클라우드 컴퓨팅은 데이터 저장을 자유롭게 해주고, 높은 성능의 컴퓨팅, 오픈 소스, 최신 툴과 라이브러리를 사용할 수 있게 해주었다.  
머신러닝은 알고리즘을 이용해 모델을 만들고 그를 통해 미래를 예측할 수 있게 해준다.  
빅데이터는 기존의 방법으로는 처리할 수 없는 거대한 데이터를 새로운 툴과 기술을 이용해 처리하는 방법을 제공한다.

## 3. 데이터의 핵심 직군

이러한 데이터 생태계에서 핵심 역할을 담당하고 있는 직군을 살펴보자.

>- 데이터 엔지니어
>- 데이터 분석가
>- 데이터 사이언티스트
>- 비즈니스 분석가
>- BI

모두 데이터 환경에서 중요한 역할을 담당하고 있지만 현재 우리가 집중적으로 살펴볼 직군은 데이터 엔지니어이다<span style="color:red">(데이터 직군에 대한 간단한 정리를 보고 싶다면 [데이터 분야의 직군 소개](https://github.com/Team-Neighborhood/I-want-to-study-Data-Science/wiki/%EB%8D%B0%EC%9D%B4%ED%84%B0-%EB%B6%84%EC%95%BC%EC%9D%98-%EC%A7%81%EA%B5%B0-%EC%86%8C%EA%B0%9C)를 참고).</span>

데이터 분석의 모든 과정은 데이터 엔지니어로부터 시작한다고 해도 과언이 아니다. 데이터 엔지니어는 데이터 구조를 설계하고 유지하며, 데이터를 이용가능하게 만들어 주는 역할을 한다. 데이터 엔지니어가 하는 일을 간략히 정리해보자.

>- 분산된 데이터를 추출, 통합, 정리
>- 데이터 정제, 변환, 준비
>- 데이터를 디자인, 저장, 관리

데이터 엔지니어가 가져야 할 필요역량을 간략하게 다음과 같이 정리해볼 수 있다. 역량은 필수적이고 도구들이 상당히 많아보이는데 모두 알아야하는 것은 아니고 몇 가지를 쓰임새에 맞게 선택하여 사용한다고 생각하면 된다.

>- 프로그래밍 역량
>- 시스템 설계 지식
>- RBMS와 NoSQL에 대한 깊은 이해

## 4. 데이터 엔지니어링이란?

이제 본격적으로 데이터 엔지니어링에 대해 살펴보자. 데이터 엔지니어링을 간략히 설명하면 데이터 흐름과 접근에 대한 정비라고 할 수 있다. 데이터 인프라 및 플랫폼을 설계, 구축, 유지 관리하는 작업라고 할 수 있다. 그래서 흔히 데이터 엔지니어를 데이터 정비공, 데이터 배관공으로 부르고는 한다. 이러한 데이터 인프라에는 데이터베이스, 빅 데이터 저장소 및 이러한 시스템 간의 데이터 변환 및 이동을 위한 데이터 파이프라인이 포함될 수 있다. 데이터 엔지니어는 데이터 시스템 개발 및 최적화 작업을 수행하고 데이터를 분석에 사용할 수 있도록 하는 사람이다. 그들의 목표는 정확한 데이터 분석을 위해 데이터를 이용가능한 퀄리티로 만드는 것이라고 할 수 있다. 

데이터 엔지니어링은 다음과 같은 과정을 가지고 있다.

<img src="https://drive.google.com/uc?export=view&id=1OUcz2YsMVbJ1HFxqdyrwYl_Q9ADjT_tO">

데이터 수집은 흩어져있는 데이터 소스를 추출하고 통합하고 정리하는 것을 말한다.

데이터 처리는 수집된 데이터를 정제하고 변환하고 사용가능하게 준비한다. 이를 위해 큰 스케일의 데이터 처리를 위한 분산 시스템, 데이터 저장소로 데이터를 추출, 변환, 로딩하기 위한 파이프라인, 데이터의 보안, 데이터의 신뢰, 확장을 위한 툴과 시스템, 워크플로우 등이 필요하다.

데이터 저장은 데이터에 대한 신뢰성과 접근성을 담당한다. 이 과정에서 중요한 것은 처리된 데이터 저장을 위한 데이터 저장소, 확장가능한 시스템, 데이터의 보안, 백업, 복구에 대한 보장성 등이 중요하다.

마지막으로 데이터를 사용자가 안전하게 사용할 수 있게 만들어주는 과정이 있다. 이 과정에서 최종 사용자를 위한 API, 서비스, 프로그램을 만들고, 사용자가 인터페이스와 대시보드를 사용할 수 있게 된다.
