## MVC 패턴



### UML 클래스 다이어그램이란?

- 객체지향 설계와 구현을 지원하기 위해 만들어진 일종의 모델링 언어(다이어그램 형태로 정의한 규격) 



### 추상 팩토리 패턴이란?

- Factory(팩토리)란 디자인 패턴에서 객체를 생성하는 역할을 의미한다

- Abstract(추상) class는 객체는 생성한 뒤 구체적인 구현은 하위 객체에 위임(오버라이드)하는 모델 

  ![Abstract_factory_UML](https://upload.wikimedia.org/wikipedia/commons/9/9d/Abstract_factory_UML.svg)

### MVC 패턴이란?

1.  Model-View-Controller의 약어로 GUI기반의 애플리케이션 개발에 사용되는 디자인패턴
2.  화면과 데이터 처리를 분리하여 코드 간 종속성을 줄임



![Router-MVC-DB](https://upload.wikimedia.org/wikipedia/commons/5/53/Router-MVC-DB.svg)

<ref style="font-size:0.7em;">출처 : https://ko.wikipedia.org/wiki/</ref>

3. 모델 
   - 데이터를 처리하는 영역 
   - 데이터베이스를 연동하는 DAO클래스와 데이터 구조를 표현하는 DO클래스로 구성됨
4. 뷰 
   - 화면을 담당하는 영역
   - 주어진 데이터를 출력
   - jsp 같은 뷰 템플릿 엔진 사용 
   - 모델, 컨트롤러와 종속성이 없도록 구현해야한다.
5.  컨트롤러
   - 사용자의 요청을 받아 데이터베이스와 연동 등의 작업을 처리한 다음 결과에 따라 페이지로 전환해주는 기능을 담당하는 것. 따라서 뷰와도 종속적인 관계임
   - 서블릿만으로 대규모 프로젝트에서 컨트롤를 구현하기 어렵기 때문에 스프링 프레임워크와 같이 오픈소스 프레임워크를 사용하게 된다.
