### 1. 데이터베이스 시스템의 개념

#### 데이터베이스 시스템(database system)

* 데이터베이스를 이용하여 데이터를 저장하고 처리하는 시스템
* 데이터베이스 시스템은 **데이터베이스**와 **데이터베이스 관리 시스템**(DBMS)로 구성되어 있다.
<br>

  ##### 데이터베이스
  * 조직이 관리하는 데이터를 한 곳에 체계적으로 모아 놓은 것

  ##### DBMS
  * 모아 놓은 데이터를 관리하고 사용자의 요구를 처리해주는 소프트웨어

  ##### 데이터베이스 서버
  * 데이터베이스/DBMS가 설치된 컴퓨터
  * 인터넷과 같은 네트워크에 연결되어 수많은 사용자들의 요구를 처리한다.
  * 다수의 사용자들이 이용하려는 목적이 크므로 성능이 좋은 컴퓨터에 설치한다.

  ##### SQL
  * 응용 프로그램이 DBMS에게 요구사항을 전달하기 위해 사용하는 표준화한 언어
  * DBMS는 응용 프로그램에게 SQL 언어를 통해 요청 사항을 접수하고, 결과를 다시 응용 프로그램에게 전달한다.

### 2. 데이터베이스와 데이터베이스 관리 시스템

#### 데이터베이스(database)

* 여러 사람이 공유할 목적으로 연관된 데이터를 체계적으로 모아 놓은 저장소 (모아 놓은 데이터의 덩어리)
* 물리적인 저장장치인 **하드디스크** 상에 구현된다.
* 오라클의 경우 여러 개의 파일들이 모여서 하나의 데이터베이스로서 역할을 한다.
<br>

  ##### 테이블(table)
    * 데이터베이스에 정보를 저장하는 기본 단위
    * 데이터베이스는 테이블들의 집합
  ##### 시스템 카탈로그(system catalog) or 데이터 사전(data dictionary)
    * 데이터베이스 안에는 사용자가 만든 테이블도 있지만, 존재하는 데이터베이스들을 관리하기 위한 정보를 포함하는 테이블도 있다. 어떤 데이터베이스에 어떤 파일들이 있는지, 각 파일의 구조는 어떠한지, 사용자는 누가 있는지 등의 정보를 관리하기 위해 만들어진 테이블들을 뜻한다.
    * 시스템 카탈로그는 DBMS가 이용하는 정보이다.
    * 데이터베이스를 생성하면 시스템 카탈로그도 함께 생성된다.
    * 따라서 데이터베이스에는 사용자가 이용할 목적으로 사용자가 생성한 테이블과 DBMS가 사용할 목적으로 자동 생성하는 시스템 카탈로그가 존재한다.
<br>
#### 데이터베이스 관리 시스템(database management system)
* 데이터베이스를 관리하는 소프트웨어
* 사용자의 요구를 언제라도 수행하기 위해 백그라운드 프로세스 형태로 상시 실행된다.
* DBMS는 역할과 기능이 복잡하기 때문에 여러 소프트웨어들이 함께 수행되며 DBMS의 역할을 수행한다.<br>

  ##### 사용자 입장에서 본 DBMS가 제공하는 3가지 서비스
  * **데이터 정의 기능**
    * 사용자가 데이터베이스를 생성하거나 데이터베이스 내에 원하는 구조의 파일(테이블)을 생성/변경할 수 있도록 지원한다.
  * **데이터 조작 기능**
    * 사용자가 데이터베이스 내의 파일(테이블)에 대해 조회하거나 수정, 삭제하는 기능을 제공한다.
  * **데이터 제어 기능**
    * 다수의 사용자가 이용하는 데이터베이스 내의 데이터를 정확하고 안전하게 유지하는 기능
    * 권한이 있는 사용자에 대해서만 데이터베이스로의 접근을 허용한다.
    * 다수 사용자의 요구를 동시에 처리하는 기능을 제공한다.(병행수행 제어) 
    * 정기적으로 데이터베이스를 백업하고, 데이터베이스에 장애가 발생하면 이를 복구하는 기능을 제공한다.

### 3. 오라클 데이터베이스 개요

#### CDB와 PDB
* 오라클 12c 버전부터 소개된 개념

  ##### PDB(Pluggable DB)
  * 일반적인 데이터베이스
  * 쉽게 꽂았다 뽑았다 할 수 있는 DB

  ##### CDB(Container DB)
  * 여러 PDB들을 담을 수 있는 더 큰 데이터베이스
  * PDB를 하나의 컨테이너로 보고, 이런 컨테이너들을 저장하는 창고의 역할을 한다.

  ##### 멀티 테넌트(Multi-Tenant) DB
  * 여러 PCB들이 CDB에 새들어 사는 것 같이 여러 데이터베이스를 운영하는 방식

#### 테이블스페이스(tablespace)
* 오라클에서는 데이터베이스의 데이터를 확장자가 .dbf로 끝나는 물리적 데이터 파일에 저장한다.
* 이러한 데이터 파일 하나 이상 모여서 테이블스페이스라는 논리적 공간을 형성한다.
* 여러 테이블스페이스가 모여 데이터베이스를 구성한다.
* 데이터베이스 $\supset$ 테이블스페이스 $\supset$ 데이터 파일
