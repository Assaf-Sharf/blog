## Git - add

Almacena todos los archivos modificados del **(Working Tree)** dentro del estado **{Staging Area}** llamado también **{INDEX}** , lo hace de forma temporal a la espera de ser enviados al **[Repositorio Remoto]** después de hacer un ``git commit -m "mensaje"`` confirmar sus cambios y hacer a continuación un ``git push``

```git
git add <archivo>
```

* Añadir un ``<archivo>`` al **{Staging Area/INDEX}** a la espera de ser confirmado mediante un ``git commit -m "mensaje"`` para su envio al «Repositorio Remoto»

```git
git add .
```

* Todos los archivos modificados del **(Working Tree)** son almacenados temporalmente en el **{Staging Area}** llamado también **{INDEX}**

***


* Agregar archivos de forma interactiva

```git
git add -i
```
