# 프로젝트 시작

프로젝트를 진행하면서 했던 생각들을 정리하기 위해 기록을 시작하기로 했다.<br>


## 주제 선정

먼저 주제를 선정하는 것이 가장 어려웠다.. 기본적으로 CRUD 기능을 구현하면서 독창적으로 다른 기능들을 메인으로 넣고 싶었다.<br>
창의적인 아이디어를 바란건 아니고, 출시하게 되면 사람들이 이용할만 하거나, 혹은 출시되지 않더라도 혼자 필요할 때 사용할 수 있었으면 좋겠다는 생각을 했다.<br>
그러다 문득 저번에 학교 동창들을 만났을 때 식당을 찾으면서, 동일한 음식을 파는 식당이 2곳인데, 어느곳이 더 좋은지 비교하느라 시간을 허비했던 적이 있었다. 여기에서 영감을 얻어서 주제를 선정할 수 있었다.

## 그렇다면 다음은?

주제를 선정했지만, 바로 뭘 해야할지는 감이 잡히지 않아서 일단 노트에 넣을 기능들을 생각하면서 화면을 스캐치했다. 그리고 취준하는 입장이라 포트폴리오에 넣을겸 WBS도 같이 만들었다.<br>
대충 기능을 추가하면서 화면을 구성하니 어느정도 그림이 그려졌고, 다음으로는 흐름도를 작성하여 어떤 화면을 더 추가할지 구상했다.

## 화면 제작

화면 스캐치를 어느정도 했기 때문에 이제 화면을 제작해야 하는데, 그동안 백엔드 쪽으로 스프링과 JPA만 공부해서 프론트단인 HTML, CSS는 미숙했다..<br>
그래도 전에 HTML, CSS 클론 코딩한게 있어서 코드를 다시 들여다 보고, 간단한 문법만 다시 복습해서 제작에 들어갔다.<br>
간단한 화면 뼈대만 만들고 디자인은 제일 마지막에 할 생각이라 금방 할 줄 알았는데 아니었다..<br>
