## static 필드 

1. 인스턴스 변수

```
package lesson02;

public class MyClass {
	private int nMyField = 0; 
	
	public void setMyField(int nNewField) {
		nMyField = nNewField;
	}
	public int getMyField() {
		return nMyField;
	}
}
```

```
package lesson02;

public class FieldTest {

	public static void main(String[] args) {
		MyClass myObject1, myObject2;
		myObject1 = new MyClass();
		myObject2 = new MyClass();
		
		myObject1.setMyField(10);
		myObject2.setMyField(100);
		
		System.out.println(myObject1.getMyField());
		System.out.println(myObject2.getMyField());
	}

}
```

myObject1.nMyField와  myObject2.nMyField는 서로 다른 필드 변수이다. 이러한 필드 변수를 인스턴스 변수라고 한다. 



2. 클래스 변수 

   static필드는 non-static필드와 달리 여러 개의 객체를 생성하더라도 해당 클래스에서 하나만 존재하며 여러 객체가 공유하는 필드 변수이다. 이러한 필드 변수를 클래스 변수라고 한다. 

   static필드는 특정 객체가 배타적으로 소유하지 않는다. 따라서 객체를 생성하지 않았더라도 클래스 이름을 통해 사용할 수 있다. 

   ```
   public class MyClass {
   	public static int nStaticField = 0; //접근제어자가 public!! 
   }
   ```

```
public class StaticTest1 {

	public static void main(String[] args) {
		// TODO Auto-generated method stub
        MyClass myOject1, myOject2;
        myOject1 = new MyClass();
        myOject2 = new MyClass();
        
        myOject1.nStaticField = 10;
        System.out.println(myOject2.nStaticField); 
        myOject2.nStaticField += 130;
        System.out.println(myOject1.nStaticField); 
        
        //클래스이름.클래스변수
        MyClass.nStaticField = 300;
       
	}
}
```

```
//결과 
10
140
```



3. static필드 사용예 

   여러 소스코드에서 상추처럼 공통되는 값으로 사용되어야 하는 값들이 있다면 이런 값을 클래스에서 static 필드로 정의해 놓고 활용할 수 있다. 

   ```
   package staticLesson;
   
   public class MyMessages {
   	public static String msg1 =  "Hello";
   	public static String msg2 =  "Java";
   	public static String msg3 =  "Application";
   	public static String msg4 =  "Applet";
   }
   ```

   ```
   package staticLesson;
   
   public class StaticTest3 {
   	public static void main(String[] args) {
   		System.out.println(MyMessages.msg1);
   		System.out.println(MyMessages.msg2);
   		System.out.println(MyMessages.msg3);
   		System.out.println(MyMessages.msg4);
   	}
   }
   ```

   

4. static 필드 주의할 점 

   static 필드에 대해 주의할 점은 초기화이다. nStaticField1필드는 필드 선언문에서 초기화가 되었고, 값이 누적된다. 생성자에서 초기화된 nStaticField2 필드는 객체가 생성될 때마다 초기화가 진행되기 때문에 값이 누적되어 변화하지 않는다. 

   

   ```
   public class MyClass2 {
   	public static int nStaticField1 = 10;
   	public static int nStaticField2;
   	
   	public MyClass2() {
   		nStaticField2 = 20;
   	}
   }
   ```

```
package staticLesson;

public class InitStatic {

	public static void main(String[] args) {
		// TODO Auto-generated method stub
       MyClass2 mc1 = new MyClass2();
       System.out.println("mc1 is constructed.");
       System.out.println("nStaticField1 : " + mc1.nStaticField1);
       System.out.println("nStaticField2 : " + mc1.nStaticField2);
       
       System.out.println("");
       
       MyClass2.nStaticField1 += 10;
       MyClass2.nStaticField2 += 10;
       
       System.out.println("");
       System.out.println("Values are changed");
       System.out.println("nStaticField1 : " + mc1.nStaticField1);
       System.out.println("nStaticField2 : " + mc1.nStaticField2);
       
       System.out.println("");
       MyClass2 mc2 = new MyClass2();
       
       System.out.println("mc1 is constructed.");
       System.out.println("nStaticField1 : " + mc1.nStaticField1);
       System.out.println("nStaticField2 : " + mc1.nStaticField2);
       
	}

}
```

```
//결과
mc1 is constructed.
nStaticField1 : 10
nStaticField2 : 20


Values are changed
nStaticField1 : 20
nStaticField2 : 30

mc1 is constructed.
nStaticField1 : 20
nStaticField2 : 20
```



## static 메소드

static 메소드는 객체와 무관하게 클래스 이름만으로 호출할 수 있으며 클래스 메소드라고 한다.

<클래스 이름>.<메소드 이름>

static 메소드는 static 필드를 제어하는 기능을 담당하도록 구현하는 것이 일반적이다. 

static 메소드는 non-static메소드를 호출할 수 없고 non-static 필드를 참조할 수 없다.
