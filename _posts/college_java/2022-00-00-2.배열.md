## 배열의 선언

1. 배열은 선언한 뒤 초기화나 생성의 과정을 거쳐야만 사용할 수 있다.

2. 배열을 선언할 때 배열의 크기를 지정할 수 없음

3. 다차원 배열은 255차원까지 선언할 수 있다. 

   ```
   /*1차원 배열의 선언*/
   int anArray1[];
   int[] anArray2;
   int anArray3[10]; //오류, 선언할 때 크기를 명시할 수 없다. 
   anArray1[0] = 1; //오류, 초기화나 생성전에 배열을 사용할 수 없다. 
   
   /*2차원 배열의 선언*/
   int anArray4[][];
   int[][] anArray5;
   int[] anAnrray6[]; //오류는 아니지만 좋지 않은 표현이다.
   ```

## 배열의 초기화 및 생성

배열의 초기화란 배열을 선언하고 초기값을 지정하는 것을 말한다. 

배열의 생성이란 배열이 사용할 메모리 공간을 확보하는 것을 말한다. 

배열을 초기화 시키면 자동으로 배열이 사용할 메모리 공간을 할당받게 된다.

 생성되지 않은 배열을 초기화 시키고자할 경우에는 반드시 선언문에서 초기화 시켜야한다.

- 배열의 초기화

```
public class ArrayTest2 {

	public static void main(String[] args) {
		// TODO Auto-generated method stub

		/*1차원 배열의 초기화*/
		
		int anArray1[] = {1,2,3,4,5};
		int[] anArray2 = {1,2,3,4,5};
		
		/*2차원 배열의 초기화*/
		int anArray3[][] = {{1,2,3},{4,5,6}};
		int[][] anArray4 = {{1,2,3},{4,5,6}};
		
		/*주의사항*/
		int anArray5[]; 
		anArray5 = {1,2,3,4,5}; //오류, 선언과 동시에 초기화되어야한다. 
	}
}
```



- 배열의 생성

  배열의 생성은 초기화 없이 단지 메모리 영역만 할당해주는 것을 말한다.

  생성된 배열은 따로 초기화 하지 않아도 원소가 숫자형인 경우에는 0또는 0.0으로, 참조형일 경우에는 null로 초기화된다. 

```
package college_java;

public class ArrayTest3 {

	public static void main(String[] args) {
		// TODO Auto-generated method stub

		/*배열의 생성은 초기화 없이 단지 메모리 영역만 할당해주는 것을 말한다.*/
		
		/*1차원 배열의 생성*/
		
		int anArray1 [] = new int[5];
		int[] anArray2 = new int[5];
		
		int anArray3[];
		anArray3 = new int[5];
		
		/*2차원 배열의 생성*/
		int anArray4[][] = new int[5][1]; 
		int[][] anArray5 = new int[5][1]; 
		
	}
}
```
