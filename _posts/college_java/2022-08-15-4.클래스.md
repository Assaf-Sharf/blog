## 클래스 정의하기

```
<접근 제어자> class <클래스 이름> {

	<필드 선언>;

	<메소드 선언>;

}
```

<접근 제어자> 

- 접근 제어자의 생략 : 같은 패키지에 있는 다른 클래스에서 사용 가능(다른 패키지는 불가능)
- public :   어디에서나 사용 가능 
- private : 중첩 클래스에서 내부 클래스를 정의할 때만 사용 가능, private로 정의된 내부 클래스는 그 것을 포함하고 있는 클래스에서만 사용가능
- protected : 중첩 클래스에서 내부 클래스를 정의할 때만 사용 가능, 내부 클래스를 포함하고 있는 클래스, 같은 패키지, 내부 클래스를 포함하고 있는 클래스를 상속받는 서브 클래스에서 사용 가능 

<필드 선언> 

- 멤버 필드, 멤버 변수, 인스턴스 변수 또는 간단히 필드라고 함 

- 개별 객체들의 상태 정보를 저장

- 클래스 정의 블록에는 필드나 메소드 정의 이외에 다른 명령문은 넣을 수 없다.

  ```
  public class Electronics {
  	
  	private int nVoltage = 0; //필드선언 private
  	
  	nVoltage = 1;             // 오류 
  	
  	public void turnOn(int nInputVoltage) {
  		nVoltage = nInputVoltage;
  	}
      public void turnOff() {
      	nVoltage = 0;
  	}
      public int getVoltage() {
      	return nVoltage;
      }
  
  }
  ```

  필드의 지정문을 메소드 외부에 넣을 수 없다. 

  ```
  {nVoltage = 1;}   
  ```

  지정문을 중괄호 블록 안에 두면 클래스 내부에 들어갈 수 있음. 이러한 블록을 초기화 블록이라고 함 

- 클래스의 필드는 외부로부터 직접적인 접근을 불허하는 것이 객체지향 프로그래밍 개념에 부합하기 때문에 필드를 정의할 때 private를  사용하는 것이 좋다. -> 필드변수는 public 메소드를 통해서 private 필드의 값을  제어할 수 있도록 한다. 

- 필드 선언문에서의 초기화 

  ```
  package fieldLesson;
  
  public class IniTest {
    private int nValue = 0;
    private static int nValue2 = 0;
  }
  ```

  필드의 선언문과 초기화 구문을 분리하면 컴파일 오류가 발생한다. 

  ```
  package fieldLesson;
  
  public class IniTest {
    private int nValue;
    nValue = 5; //에러
    private static int nValue2;
    nValue2 = 10; //에러
  }
  
  ```
