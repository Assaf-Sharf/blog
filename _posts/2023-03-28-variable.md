# 변수

변수는 프로그래밍 언어에서 데이터를 관리하기 위한 핵심 개념이다.

## 1. 변수는 왜 필요할까?

아래 코드를 보자

```javascript
10 + 20;
```

사람은 계산과 기억을 모두 두뇌에서 하지만, 컴퓨터는 연산과 기억을 수행하는 부품이 나눠져 있다. 컴퓨터는 CPU를 사용해 연산하고, 메모리를 사용해 데이터를 기억한다.

위 코드의 숫자 값 10과 20은 메모리 상의 임의의 위치(메모리 주소)에 기억(저장)되고 CPU는 이 값을 읽어들여 연산을 수행한다. 연산 결과로 생성된 숫자 값 30도 메모리 상의 임의의 위치에 저장된다.

성공적으로 연산이 끝났고 연산 결과도 메모리에 저장되었지만 문제가 있다. CPU가 연산해서 만들어낸 숫자 값 30을 재사용할 수 없다는 것이다.

만약 연산 결과 30을 재사용하고 싶다면 메모리 주소를 통해 연산 결과 30이 저장된 메모리 공간에 직접 접근하는 것 외에는 방법이 없다.

하지만 <mark>메모리 주소를 통해 값에 접근하는 것은 치명적 오류를 발생시킬 가능성이 높은 매우 위험한 일</mark>이다. 따라서 <ins>JavaScript는 개발자의 직접적인 메모리 제어를 허용하지 않는다.</ins>

따라서 프로그래밍 언어는 기억하고 싶은 값을 메모리에 저장하고, 저장된 값을 읽어 들여 재사용하기 위해 **변수**라는 메커니즘을 제공한다.

**<mark>변수<sup>variable</sup>는 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름을 말한다.</mark>**

간단히 말해 변수는 프로그래밍 언어에서 값을 저장하고 참조하는 메커니즘으로, **<mark>값의 위치를 가리키는 상징적인 이름</mark>**이다.

앞서 살펴본 Code를 변수<sup>variable</sup>를 사용해 다시 작성해보면

```javascript
var result = 10 + 20;
```

메모리 공간에 저장된 값을 식별할 수 있는 고유한 이름(위 Code에서는 result)을 **변수 이름**(또는 변수명)이라 한다.

그리고 변수에 저장된 값(위 예제에서는 30)을 **변수 값**이라고 한다.

## 2. 식별자

변수 이름을 식별자<sup>identifier</sup>라고도 한다. **<mark>식별자는 어떤 값을 구별해서 식별할 수 있는 고유한 이름을 말한다.</mark>**

위에 코드에서 식별자 result는 값 30을 식별할 수 있었다. 이를 위해 식별자 result는 값 30이 저장되어있는 메모리 주소를 기억해야 한다.

**<mark>식별자는 값이 아니라 메모리 주소를 기억하고 있다.</mark>** 식별자로 값을 구별해서 식별한다는 것은 식별자가 기억하고 있는 메모리 주소를 통해 메모리 공간에 저장된 값에 접근할 수 있다는 의미다. 즉, <ins>식별자는 메모리 주소에 붙인 이름이라고 할 수 있다.</ins>

## 3. 변수 선언의 실행 시점과 변수 호이스팅

```javascript
console.log(score); // undefined

var score; // 변수 선언문
```

위 Code 결과를 예상해보자 변수 선언문보다 변수를 참조하는 코드가 앞에 있다. JavaScript는 한 줄씩 순차적으로 실행되기에 참조 에러<sup>ReferenceError</sup>가 발생할 것처럼 보이지만 undefined가 출력된다.

그 이유는 **<mark>변수 선언이 소스코드가 한 줄씩 순차적으로 실행되는 시점, 즉 런타임<sup>runtime</sup>이 아니라 그 이전 단계에서 먼저 실행되기 때문이다.</mark>**

<p style='background-color: lightgrey; color: black;'>자바스크립트 엔진은 소스코드를 한 줄씩 순차적으로 실행하기에 앞서 먼저 소스코드의 평가 과정을 거치면서 소스코드를 실행하기 위한 준비를 한다. 이 때 소스코드 실행을 위한 준비 단계인 소스코드의 평가 과정에서 자바스크립트 엔진은 변수 선언을 포함한 모든 선언문(변수 선언문, 함수 선언문 등)을 소스코드에서 찾아내 먼저 실행한다. 그리고 소스코드의 평가 과정이 끝나면 비로소 변수 선언을 포함한 모든 선언문을 제외하고 소스코드를 한 줄씩 순차적으로 실행한다. </p>

이처럼 **<mark>변수 선언문이 코드의 선두로 끌어 올려진 것처럼 동작하는 자바스크립트 고유의 특징을 <span style='color: red'>변수 호이스팅<sup>variable hoisting</sup></span>이라 한다.</mark>**

## 4. 값의 재할당

재할당이란 이미 값이 할당되어 있는 변수에 새로운 값을 또다시 할당하는 것을 말한다.

```javascript
var score = 80; // 변수 선언과 값의 할당
score = 90; // 값의 재할당
```

var 키워드로 선언한 변수는 값을 재할당할 수 있다. var 키워드로 선언한 변수는 선언과 동시에 undefined로 초기화되기 때문에 엄밀히 말하자면 <ins>변수에 처음으로 값을 할당하는 것도 사실은 재할당이다.</ins>

만약 **값을 재할당할 수 없어서 변수에 저장된 값을 변경할 수 없다면 변수가 아니라 <ins>상수<sup>constant</sup></ins>라 한다.**

상수는 한번 정해지면 변하지 않는 값이다.

변수에 값을 재할당하면 score 변수의 값은 이전 값 80에서 재할당한 값 90으로 변경된다. 이전 값 80이 저장된 메모리 공간을 지우고 새롭게 90을 저장하는 것이 아니라 **<ins>새로운 메모리 공간을 확보하고 그 공간에 숫자 값 90을 저장</ins>**한다.
