# 잔디 채우기


블로그의 종류가 아주 많지만 git blog를 선택한 이유 중 하나가 블로그를 운영하면서 git의 잔디를 채울 수 있어서인 사람은 아주 많을 것이다.<br>
하지만 fork를 통해 생성된 git 블로그는 아무리 commit, push를 해도 잔디가 채워지지 않는다.

먼저 fork로 생성한 git 블로그는 왜 잔디가 채워지지 않는지 알아볼 필요가 있다.

git에서 기여로 간주되는 항목은 다음과 같은데 <br>
![image](https://github.com/user-attachments/assets/0c83df16-a453-4159-90b2-b27fd337eace)

여기에 첫번째 줄에 적힌 commit을 하면 잔디가 생긴다는데 왜 git 블로그는 commit을 해도 잔디가 심어지지 않는지 의문이 생길 수 있다.



이 부분에 대한 해답은 fork 관련 문서에서 찾을 수 있는데 포크로 생성된 repository에서는 pull request를 통해 기여를 할 수 있다고 나온다. <br>
![image](https://github.com/user-attachments/assets/5892306f-81fe-49f9-a18f-2f8bfe9f600c)


또한 pull request는 한 branch의 변경 내용을 다른 branch와 병합을 하는 것이여서 새로운 branch를 만들어서 기존 branch에 pull request를 하면 기여가 가능할 것으로 보인다.<br>
![image](https://github.com/user-attachments/assets/2588df0d-23c6-4e52-8218-b3bc7acfc133)


pull request를 하기 위해 우선 branch를 새로 만든다.<br>
