# AutoLayout

tip 

- 많이 이것저것 만져봐야한다
- 기준
- Priority

 

먼저 A라벨에 왼쪽과 위쪽에 24만큼의 제약을 둔다

B 라벨에 오른쪽과 위쪽에 24만큼의 제약을 둔다

여기서 A 를 B에 Leading을 걸어 준다면 아래와 같이 오류가 난다.

A B 둘다 Priority 는 같다.

![스크린샷 2021-11-05 오전 1.20.21.png](Hoon's%20349d9560bc3e48cd8005e558d80475f8/%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2021-11-05_%EC%98%A4%EC%A0%84_1.20.21.png)

오류가 나는 이유는?

먼저 Label은 글자만큼(?)의 크기를 갖는다.

그렇기 때문에 현 상태에서 길이 만큼의 폭을 기본적으로 갖는데 

제약을 주는데 서로의 우선순위가 같기 때문에 먼저 어느 곳에 맞춰야 할 기준이 없기 때문

**기준이 모호하기 때문에 어떠한 제약도 걸리지 않는다

***제약을 두는 순서는 중요하지 않다.

해결 방안은 많다.

~~**어느것이 맞다 할 수 없다.!!!!**~~

Leading의 Priority값을 낮춰준다.

후

Leading의 Priority가 낮기 때문에 Priority가 같은 A B의 레이아웃은 고정된다.

두번째로는

A label의 Priority를 높여준다.

![스크린샷 2021-11-05 오전 1.30.43.png](Hoon's%20349d9560bc3e48cd8005e558d80475f8/%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2021-11-05_%EC%98%A4%EC%A0%84_1.30.43.png)

왜? 오류가 해결될까?

A Label의 Priority의 값을 높여줌으로

B의 우선순위가 낮아지기 때문에

A를 B에 Leading을 걸어 주었으나

 B가 Priority가 낮기 때문에 딸려오게 된다.

하지만 의도가 A 를 B에 Leading을 걸어주어 A를 고정된 B 에 맞추려는 것이라면 반대로 B의 Priority를 높이거나 A의 Priority를 낮추면된다

![스크린샷 2021-11-05 오전 1.34.17.png](Hoon's%20349d9560bc3e48cd8005e558d80475f8/%EC%8A%A4%ED%81%AC%EB%A6%B0%EC%83%B7_2021-11-05_%EC%98%A4%EC%A0%84_1.34.17.png)

이렇게 B의 Priority 가 높기 때문에 A가 딸려오게 된다.

이처럼 어떠한 Constrain을 가지느냐에 따라 Layout은 천차만별이다.

# 옵셔널

옵셔널은 왜쓸까?

지긋지긋하게 책에서... 유튜브에서.. 강의에서..

옵셔널은 굉장히 영리하다.

뭐가?

단순히 코딩 연습에서는 모두 정해진 값을 쓰기 때문에 모른다.

서버에서 사이트에서 데이터를 받아오거나, 본인의 프로젝트 내에서 옵셔널 값을 사용함으로

nil 값 즉, 아무런 값을 받아오지 못했을 때를 대비할 수 있다.

날씨 앱에서 그날 기상청에서 날씨에 관한 데이터를 업로드 하지 않았다던가 나중에 추가 하기 위한 값을 비워두기 위해서도 사용하며 다양한 곳에서 사용 될 수 있다.

바인딩이나 옵셔널은 책에 충분히 나온다.

또한 삼항 연산자를 사용한예시를 보면

```swift
private func changeBackgroundColorBtn(color: UIColor) {
        self.blackBtn.alpha = color == UIColor.black ? 1 : 0.2
        self.blueBtn.alpha = color == UIColor.blue ? 1 : 0.2
        self.orangeBtn.alpha = color == UIColor.orange ? 1 : 0.2
    }
```

이는 

        self.blackBtn.alpha = color == [UIColor.black](http://UIColor.black) ? 1 : 0.2
여기서 (blackBtn의 알파 값을,,) color 가 UIColor.black 이면? 1로바꿔라, 아니면 0.2로 바꿔라!

라고 해석할 수 있다.
